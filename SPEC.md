## File Format

Template files use the `.tmpl.md` extension. They are valid markdown
files with an optional YAML frontmatter block followed by a body:

```markdown
---
<frontmatter>
---

<body>
```

**Standalone control-flow tags** (`{% %}`) on their own line must carry a `>`
blockquote prefix. Content lines (text, `{{ }}` expressions) do not need it.
The prefix is stripped before compilation and has no effect on output. In a
markdown editor it renders as a blockquote, visually separating logic from prose:

<!-- prettier-ignore -->
```markdown
> {% if condition %}
Some prose.
> {% /if %}
```

**Inline tags** on a single line work without any prefix:

```markdown
{% if x %}yes{% else %}no{% /if %}
```

## Frontmatter & Type System

```yaml
---
name: my_template # optional display name
description: A summary # optional description
params:
  - name = str
  - count = int
  - score = float := 0.95 # optional default value
  - active = bool := true
  - items = list<label = str, score = int>
  - config = dict<timeout = int, retries = int>
  - status = enum<Active, Paused, Stopped>
  - outcome = enum<Confirmed(evidence = str), Rejected>
allow_unused: false # default; set true to suppress unused-param errors
---
```

### Type Reference

| Annotation                  | Rust equivalent (generated by `template_params_struct!`) | Notes                                       |
| --------------------------- | -------------------------------------------------------- | ------------------------------------------- |
| `str`                       | `String`                                                 |                                             |
| `bool`                      | `bool`                                                   |                                             |
| `int`                       | `i64`                                                    |                                             |
| `float`                     | `f64`                                                    |                                             |
| `list<field = type, ...>`   | `Vec<StructName>`                                        | Each field is a typed dict field            |
| `dict<field = type, ...>`   | Nested generated struct                                  |                                             |
| `enum<Variant1, Variant2>`  | Generated enum                                           | No payload variants                         |
| `enum<V(field = type), V2>` | Generated enum with struct variants                      | Fields accessible inside `{% match %}` arms |

All parameters **must** have explicit types. Bare `- name` without a type is a
hard error.

### Default Values

Append `:= <literal>` after the type:

```markdown
- name = str := "World"
- count = int := 42
- verbose = bool := false
- threshold = float := 0.95
```

Defaults are type-checked at compile/parse time. If a param with a default is
omitted from the render context, the default is injected automatically.

Query defaults programmatically:

```rust
use prompt_templates::Template;

let tmpl = Template::from_source("---\nparams:\n  - name = str := \"World\"\n---\nHello {{ name }}!").unwrap();
let defaults = tmpl.defaults();
assert_eq!(defaults.len(), 1);

let ctx = prompt_templates::Context::new();
assert_eq!(tmpl.render(&ctx).unwrap(), "Hello World!");
```

### Unused Parameters

Declared params that are never referenced in the body (not even in a
`{# comment #}`) are a hard error by default. Disable this with
`allow_unused: true` in frontmatter, or call
`Template::from_source_allowing_unused()`.

> **Note:** Undeclared params (referenced in the body but absent from
> `params:`) are always rejected, even with `allow_unused: true`.

## Expression Syntax

Variable substitution: `{{ expr }}`

```markdown
{{ name }}
{{ bug.title }}
{{ bug.component.label }}
```

Dotted paths resolve nested dict and enum fields. Accessing a field that does
not exist on the resolved type is a compile-time error.

## Filters

Pipe operator chains transforms left-to-right: `{{ expr | filter | filter }}`

```markdown
{{ name | upper }}
{{ name | trim | lower }}
{{ score | fixed(2) }}
{{ items | join(", ") }}
```

| Filter           | Input  | Output | Description                       |
| ---------------- | ------ | ------ | --------------------------------- |
| `upper`          | str    | str    | UPPERCASE                         |
| `lower`          | str    | str    | lowercase                         |
| `trim`           | str    | str    | Strip leading/trailing whitespace |
| `fixed(N)`       | number | str    | Format with N decimal places      |
| `length`         | any    | int    | Length of list, string, or dict   |
| `default("val")` | any    | any    | Fallback if value is falsy        |
| `join("sep")`    | list   | str    | Join list items with separator    |
| `limit(N)`       | list   | list   | Take first N elements             |
| `gt(N)`          | number | bool   | True if value > N                 |

## Built-in Functions

| Function       | Returns | Description                                            |
| -------------- | ------- | ------------------------------------------------------ |
| `idx(binding)` | int     | 0-based loop index of a `for` binding                  |
| `len(expr)`    | int     | Length of a list, string, or dict                      |
| `str(expr)`    | str     | Explicit conversion to string (int, float, bool, enum) |

`idx()` tracks each loop variable independently in nested loops:

```rust
use prompt_templates::{ctx, Template};

let tmpl = Template::from_source("---
params:
  - outer = list<label = str>
  - inner = list<label = str>
---
{% for a in outer %}{% for b in inner %}{{ idx(a) }}.{{ idx(b) }} {% /for %}{% /for %}").unwrap();

let output = tmpl.render(&ctx! {
    outer: [{ label: "x" }, { label: "y" }],
    inner: [{ label: "p" }, { label: "q" }],
}).unwrap();
assert_eq!(output, "0.0 0.1 1.0 1.1 ");
```

## Control Flow

### For Loops

```markdown
> {% for bug in bugs %}

- **{{ bug.title }}**: {{ bug.description }}
  > {% /for %}
```

`{% for x in y %}` requires `y` to be a `list` type — enforced at compile time.

### Conditionals

<!-- prettier-ignore -->
```markdown
> {% if severity == "critical" %}
🔴 Immediate action required.
> {% elif severity == "high" %}
🟠 High priority.
> {% else %}
🟢 Normal.
> {% /if %}
```

Operators: `==`, `!=`, `<`, `>`, `<=`, `>=`. Plain identifiers are evaluated
for truthiness.

### Match / Case (Enums)

Dispatch on enum variants with compile-time exhaustiveness checking:

**Multi-arm** (must cover all variants):

<!-- prettier-ignore -->
```markdown
> {% match outcome %}
> {% case Confirmed %}
Confirmed with evidence.
> {% case NotConfirmed %}
Not confirmed.
> {% /match %}
```

**Multi-variant arm** (shared body for several variants):

<!-- prettier-ignore -->
```markdown
> {% case Confirmed | ConfirmedWithCaveats %}
Evidence found.
```

**Inline guard** (renders only if variant matches):

```markdown
> {% match vuln_type case Known %}({{ vuln_type.label }}){% /match %}
```

Inside a `{% match %}` arm, the variant's fields are accessible via `expr.field`
after type narrowing:

- `{% case A | B %}` — only fields present on **both** A and B are accessible.
- `{{ outcome.evidence }}` outside a `{% case Confirmed %}` is a compile error
  if `evidence` is not shared by all variants.

#### Compile-time guarantees for `match`

1. Unknown variant names → compile error.
2. Field narrowing — field access outside a matching arm → compile error.
3. Multi-variant intersection — only shared fields allowed.
4. Exhaustiveness — all variants must be covered in multi-arm match.

## Includes

```markdown
> {% include [name](path.tmpl.md) %}
> {% include [child](child.tmpl.md) with msg=greeting %}
> {% include [row](row.tmpl.md) for item in items %}
```

- The `[name]` part is a standard markdown link — clickable in editors.
- The `(path.tmpl.md)` is the file path, resolved relative to the including
  template.
- **Explicit parameter passing** via `with` is required; no implicit scope
  leaking.
- **Iterated includes** via `for` unroll the list, passing each element.
- Parameters are type-checked against the included template's frontmatter.
- Maximum nesting depth: 16 (configurable via `.with_max_include_depth(n)`).

## Inline Templates

Define reusable fragments inline, without separate files:

```markdown
## {% tmpl bug_row %}

## params:

## - title = str

## - severity = str

- **{{ title }}** ({{ severity }})
  {% /tmpl %}

> {% for bug in bugs %}
> {% include bug_row with title=bug.title, severity=bug.severity %}
> {% /for %}
```

The `##` prefix on frontmatter lines inside `{% tmpl %}` works like the `>`
blockquote prefix in files — stripped before compilation, keeps source as valid-looking markdown.

Inline templates support: typed frontmatter, `with` parameter passing, `for`
iteration, and full type checking. They are compiled once and reused at every
include site.

## Raw Blocks

Output literal template syntax without processing:

```rust
use prompt_templates::{Context, Template};

let tmpl = Template::from_source("---
params: []
---
> {% raw %}
{{ not_processed }}
> {% /raw %}").unwrap();

let ctx = Context::new();
assert_eq!(tmpl.render(&ctx).unwrap(), "{{ not_processed }}\n");
```

Custom delimiter to escape `{% /raw %}` itself:

```markdown
{% raw=DELIM %}
This outputs {% raw %}...{% /raw %} literally.
{% /DELIM %}
```

## Comments

Template comments are stripped from output. Parameters referenced in comments
count as "used" for unused-parameter analysis:

```markdown
{# This comment won't appear in output #}
{# {{ reserved_var }} — suppresses unused-parameter error #}
Hello {{ name }}!
```

## Whitespace Control

Add `-` inside any delimiter to strip adjacent whitespace:

| Delimiter | Effect                                                        |
| --------- | ------------------------------------------------------------- |
| `{%-`     | Strips whitespace _before_ the tag (back to previous newline) |
| `-%}`     | Strips whitespace _after_ the tag (through next newline)      |
| `{{-`     | Strips whitespace _before_ the expression                     |
| `-}}`     | Strips whitespace _after_ the expression                      |

```rust
use prompt_templates::{ctx, Template};

let tmpl = Template::from_source("---
params:
  - name = str
---
hello  {{- name -}}
bye").unwrap();

let output = tmpl.render(&ctx! { name: "world" }).unwrap();
assert_eq!(output, "helloworldbye");
```

## Error Diagnostics

All errors are returned as `TemplateError` variants with structured fields:

```rust
use prompt_templates::{Context, Template, TemplateError};

let err = Template::from_source("---
params:
  - x = str
---
{{ x | badfilter }}").unwrap_err();

if let TemplateError::Syntax(syn) = &err {
    assert!(syn.line.is_some());
    assert!(syn.snippet.is_some());
    assert!(syn.message.contains("badfilter"));
}
```

Type mismatches include the path to the failing field:

```rust
use prompt_templates::{VarType, VarDecl, TypeCheckError, Value};

let var_type = VarType::List(vec![VarDecl {
    name: "score".into(),
    var_type: VarType::Int,
    default_value: None,
}]);

let items = Value::List(vec![
    Value::Dict([("score".into(), Value::Int(10))].into()),
    Value::Dict([("score".into(), Value::Str("bad".into()))].into()),
]);

let err = var_type.check(&items).unwrap_err();
assert_eq!(err.path, "[1].score");
assert_eq!(err.expected, "int");
assert_eq!(err.actual, "str");
```

## Compile-Time Macros

The companion crate `prompt-templates-macros` provides proc macros that validate
templates at `cargo build` time — syntax errors, unknown variables, and type
mismatches are caught before the binary runs.

### `include_template!`

Pre-parses and validates a template at compile time. Zero parsing overhead at
runtime:

```rust
use prompt_templates_macros::include_template;

let tmpl = include_template!("prompts/simple_greeting.tmpl.md");

let mut ctx = prompt_templates::Context::new();
ctx.set("name", "world");
let output = tmpl.render(&ctx).unwrap();
assert_eq!(output, "\nHello world!\n");
```

### `template_params_struct!`

Generates a **typed parameter struct** from a template's frontmatter.
Field types, enum variants, and nested structs are all derived from the
`.tmpl.md` file:

```rust
prompt_templates_macros::template_params_struct!(
    "prompts/greeting.tmpl.md" => GreetingParams
);
// Generates approximately:
//   struct GreetingParams { pub name: String, pub count: i64, pub items: Vec<GreetingParamsItemsItem> }
//   struct GreetingParamsItemsItem { pub label: String }
//   impl GreetingParams { fn render(&self, tmpl: &Template) -> Result<String, TemplateError> }
```

**Compile-time rendering** (zero parsing at runtime):

```rust
# prompt_templates_macros::template_params_struct!("prompts/greeting.tmpl.md" => GreetingParams);
let tmpl = prompt_templates_macros::include_template!("prompts/greeting.tmpl.md");
let output = GreetingParams {
    name: "Alice".into(),
    count: 42,
    items: vec![GreetingParamsItemsItem { label: "hello".into() }],
}.render(&tmpl).unwrap();
assert!(output.contains("Hello Alice!"));
```

**Runtime hot-reload** with contract validation:

```rust
# prompt_templates_macros::template_params_struct!("prompts/greeting.tmpl.md" => GreetingParams);
let tmpl = prompt_templates::Template::from_file(
    std::path::Path::new("prompts/greeting.tmpl.md")
).unwrap();
GreetingParams::validate_template(&tmpl).unwrap();
let output = GreetingParams {
    name: "Bob".into(),
    count: 1,
    items: vec![],
}.render(&tmpl).unwrap();
assert!(output.contains("Hello Bob!"));
```

## Loading & Caching

### From a File or String

```rust
use prompt_templates::{Template, load_template, ctx};

// From an inline string
let tmpl = Template::from_source("---\nparams:\n  - name = str\n---\nHello {{ name }}!").unwrap();
assert_eq!(tmpl.render(&ctx! { name: "world" }).unwrap(), "Hello world!");
```

```rust
// From a file (requires tempfile dev-dependency in this doctest)
let dir = tempfile::tempdir().unwrap();
let path = dir.path().join("greeting.tmpl.md");
std::fs::write(&path, "---\nparams:\n  - name = str\n---\nHi {{ name }}!").unwrap();

let tmpl = prompt_templates::Template::from_file(&path).unwrap();
let tmpl2 = prompt_templates::load_template(dir.path(), "greeting").unwrap();

let mut ctx = prompt_templates::Context::new();
ctx.set("name", "world");
assert_eq!(tmpl.render(&ctx).unwrap(), "Hi world!");
assert_eq!(tmpl2.render(&ctx).unwrap(), "Hi world!");
```

### `TemplateCache`

Avoids redundant parsing during hot-reload. Files are hashed — unchanged files
return the cached compilation with zero re-parsing:

```rust
use prompt_templates::TemplateCache;

let dir = tempfile::tempdir().unwrap();
let path = dir.path().join("greeting.tmpl.md");
std::fs::write(&path, "---\nparams:\n  - name = str\n---\nHello {{ name }}!").unwrap();

let cache = TemplateCache::new();
let tmpl  = cache.load(&path).unwrap();
let tmpl2 = cache.load(&path).unwrap();
assert_eq!(tmpl.source_hash(), tmpl2.source_hash());
```

`render_cached()` resolves `{% include %}` directives through the cache,
avoiding repeated disk reads:

```rust
use prompt_templates::TemplateCache;

let dir = tempfile::tempdir().unwrap();
let path = dir.path().join("main.tmpl.md");
std::fs::write(&path, "---\nparams:\n  - title = str\n---\n{{ title }}!").unwrap();

let cache = TemplateCache::new();
let tmpl  = cache.load(&path).unwrap();
let mut ctx = prompt_templates::Context::new();
ctx.set("title", "Hello");
let output = tmpl.render_cached(&ctx, &cache).unwrap();
assert_eq!(output, "Hello!");
```

### Runtime Hot-Reload & Declaration Validation

`validate_declarations()` ensures a reloaded template's `params:` block hasn't
been mutated (e.g. by an agent editing the file at runtime):

```rust
let dir = tempfile::tempdir().unwrap();
let path = dir.path().join("my.tmpl.md");
std::fs::write(&path, "---\nparams:\n  - name = str\n---\nHello {{ name }}!").unwrap();

let original = prompt_templates::Template::from_file(&path).unwrap();
let expected_decls = original.declarations().to_vec();

let reloaded = prompt_templates::Template::from_file(&path).unwrap();
reloaded.validate_declarations(&expected_decls).unwrap();

let mut ctx = prompt_templates::Context::new();
ctx.set("name", "world");
assert_eq!(reloaded.render(&ctx).unwrap(), "Hello world!");
```
